1.1 首页Home （经营实时监控大屏）(Real-time operations dashboard)

用途：平台经营状况的实时监控大屏。Purpose: A real-time dashboard to monitor platform operations.

详细讲解Details
汇总显示今日总存款、总提款、净利润、在线人数、新增注册人数等核心指标，便于管理者快速判断盈亏与运营趋势。Shows key daily KPIs (total deposits/withdrawals, net profit, online users, new registrations, etc.) so managers can quickly assess P&L and trends.

2.1 银行列表Bank List （玩家可见本地银行类型）(Player-visible local banks)

用途：设置平台允许玩家使用的本地银行类型。Purpose: Enable/disable local bank options available to players.

详细讲解Details
在此处开启或关闭特定银行（如 Maybank、CIMB 等）。只有激活的银行，玩家在前端存款时才能看到对应选项。Turn on/off specific banks (e.g., Maybank, CIMB). Only enabled banks will appear on the frontend deposit page.

2.2 收款银行卡Bank Account （公司收款卡配置）(Company receiving accounts)

用途：配置公司用于收款的银行卡详细信息。Purpose: Configure the company bank accounts used to receive deposits.

详细讲解Details
用于玩家 Local Bank 转账。需填写开户名、账号，以及每笔订单限额。Used for Local Bank transfers. Fill in account name/number and per-order limits.

2.3 收款额度切换Bank Account Door （收款额度自动切换）(Auto rotation by limit)

用途：银行卡收款额度自动切换管理。Purpose: Automatically rotate receiving cards based on daily limits.

详细讲解Details
可设置单张收款卡每日最高收款限额；当收满设定金额，系统自动切换到下一张备用卡，实现收款分流，降低风控风险。Set a daily cap per receiving card. When the cap is reached, the system switches to the next standby card to distribute volume and reduce risk.

2.4 玩家提款银行Member Bank List

用途：维护支持玩家提款的银行。Purpose: Maintain which banks players can bind for withdrawals.

详细讲解Details
设置系统支持哪些银行作为玩家收款方；若银行不在列表中，玩家无法绑定该银行提款。Define which banks are allowed as payout destinations. If a bank is not listed, players cannot bind it for withdrawals.

2.5 三方通道银行Pay Bank List （Payment Gateway）(Payment Gateway)

用途：Payment Gateway 存款渠道配置。Purpose: Configure deposit channels for payment gateways.

详细讲解Details
对接三方支付平台（如扫码支付、快速网银）的后台开关；可配置每个三方通道支持的银行。Manage gateway switches (e.g., QR, fast online banking) and configure which banks each channel supports.

详细讲解Details

3.1 新订单审核New Transaction （核心账务审核中心）(Core approval center)

用途：所有存款、提款申请的审核入口。Purpose: Central place to review and approve deposits/withdrawals.

详细讲解Details
所有存款/提款申请汇集于此。客服核对银行流水后，点击“通过/拒绝”审批存款订单；若使用 Payment Gateway，通常为自动化。All deposit/withdrawal requests land here. Staff verify bank statements then approve/reject. With payment gateways, deposits are typically automated.

3.2 快捷审核New Transaction (Fast)

用途：快捷订单处理通道。Purpose: A simplified workflow to speed up processing.

详细讲解Details
简化版审核页面，用于提升操作效率。A streamlined approval screen designed for faster operations.

3.3 交易历史Transaction History

用途：查询所有玩家存款、提款、返水等订单记录。Purpose: Query all completed/rejected/pending orders (deposit, withdrawal, rebate, etc.).

详细讲解Details
支持按时间、单号、玩家名深度查询，是追溯账目问题的主要入口。Supports deep search by time, order number, and username—primary place to investigate finance issues.

3.4 交易报表Transaction Report （玩家维度财务日报）(Player-level daily finance)

用途：统计每位玩家每日存提频率及总额。Purpose: Daily deposit/withdraw frequency and totals per player.

详细讲解Details
用于分析大客户（VIP）的注资习惯及活跃度。Useful to analyze VIP funding behavior and activity patterns.

3.5 代理提款审核Transfer Agent WD

用途：代理提款审核页面。Purpose: Review agent commission withdrawal requests.

详细讲解Details
独立于玩家提款，专门审核下线代理的佣金提取申请，确保财务分类清晰。Separate from player withdrawals; dedicated to agent commission cash-out approvals for clear accounting.

3.6 代理交易历史Tran Agent History

用途：查看代理的存提、佣金发放及返水数据。Purpose: View agent transactions, commission payouts, and rebate data.

详细讲解Details
方便管理层核对渠道成本与代理相关资金流水。Helps management reconcile channel costs and agent cash flow.
